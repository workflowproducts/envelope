Tag:

<gs-text></gs-text>

Description:

Examples:

Skeleton Example:

Without attributes the <gs-text> will have the following behaviors:

  1. Clicking on the control gives you a cursor.
  2. You can change the text by typing.
Attribute [column]:

When you have a gs-text inside a data element and you want to use it to display cell data you use the column attribute.

Attribute [maxlength=""]:

To set the maximum length of characters, use the [maxlength=""] attribute.

Attribute [onchange=""]:

To run javascript when the value changes, use the [onchange=""] attribute.

Attribute [value=""]:

To set the default value for the <gs-text>, use the [value=""] attribute.

Attribute [encrypted="keyVariable"]:

When you have a database column that contains sensitive information use the [encrypted] attribute to encrypt the value. The column will be encrypted with AES using the global variable that matches the [encrypted] attribute. If the gs-text detects a bad password, it will emit a password-error event with the key variable in event.keyVariable and either 'bad' or 'no' in event.reason (). The password for this example is 'test'.

Attribute [qs=""]:

The name [qs] stands for "querystring". The querystring is the part in the URL after the "?". The [qs] attribute has two formats, each with their own use.

  • qs="key1=attribute1". This format allows you to fill any attribute with any key from the querystring. To use multiple keys from the querystring you can put a comma-separated list of these expressions (for example: qs="key1=attribute1,key2=attribute2"). If you need an attribute to be added when a querystring key is not present, replace the "=" with "!=" (for example: qs="key1!=attribute1", attribute1 will only be added if key1 is not in the querystring). As a shorthand, if the key in the querystring is the same name as the attribute you want to fill, you can omit the attribute name in the expression (for example: qs="key1="). If an expression doesn't find it's key in the querystring: it reverts the attribute that it fills to it's original value.
  • qs="key". This format will update the [value] attribute with the querystring whenever the querystring changes.
Attribute [placeholder]:

The [placeholder] attribute can be used when you don't want to use a label to explain the purpose of the control to the user. (The label and the placeholder can be used together, that's just not what normally happens.)

Attribute [mini]:

Sometimes you are limited on space. The [mini] attribute can be used to make the <gs-text> smaller.

Attribute [tabindex=""]:

With the [tabindex=""] attribute, you can control what order your controls are tabbed to.

Attribute [autocorrect=""]/[autocapitalize=""]/[autocomplete=""] Attribute:

If you don't want spelling to autocorrect, you can use the html5 [autocorrect="off"] attribute. See also html5 [autocapitalize="off"] and [autocomplete="off"].

Attribute [disabled]:

With the [disabled] attribute, you can control whether or not you can use the control. Disabled still allows you to copy the current text.

Attribute [readonly]:

The [readonly] attribute is similar to [disabled] because it prevents the value from changing. The difference is that readonly is styled differently.

Pass-Through Attributes:

Some attributes will pass from the <gs-text> to the <input> inside of it, these are:

  • placeholder
  • name
  • maxlength
  • autocorrect
  • autocapitalize
  • autocomplete
  • autofocus
  • spellcheck
  • readonly
