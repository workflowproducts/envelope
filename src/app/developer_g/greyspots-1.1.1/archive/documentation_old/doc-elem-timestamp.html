Timestamp

WARNING: This element is deprecated, please use the <gs-dt></gs-dt> instead

Tag:

<gs-timestamp></gs-timestamp>

Description:

The <gs-timestamp> control is the <gs-date> and <gs-time> controls combined.

When working with dates on a website we heavily prefer slash delimited dates because a dash delimited date (in some cases) will text wrap, and you want everything to be consistent so please use slash delimited dates.

Examples:

Skeleton Example:

A basic <gs-timestamp> element. Without attributes the gs-timestamp control will have the following behaviors:

  1. There is a button that activates the date picker.
  2. After choosing a date using the date picker, the date picker disappears and the data changes in the control.
  3. You can also change the date manually.
  4. When the date format is slash or dash delimited:
    1. Clicking into the date will select the closest part of the date.
    2. Arrowing up or down will increase the value of the date depending on the part you have selected.
    3. Arrowing left or right will select the next or previous part.
    4. Pressing or "/" or "-" will select the next part.
    5. Pressing any number will overwrite the number of the part you have selected.
  5. There is a button that activates the time picker.
  6. If the time picker is in modal mode: the time picker can be dismissed by clicking the "Done" button. If the time picker is not in modal mode: clicking out of the time control will dissmiss the time picker.
  7. You can also change the time manually.
Attribute [value=""]:

To set the value for the <gs-timestamp>, use the [value=""] attribute.

Attribute [column]:

When you have a gs-timestamp inside a data element and you want to use it to display cell data you use the column attribute.

Attribute [onchange=""]:

To run javascript when the value changes, use the [onchange=""] attribute.

Attribute [qs=""]:

The name [qs] stands for "querystring". The querystring is the part in the URL after the "?". The [qs] attribute has two formats, each with their own use.

  • qs="key1=attribute1". This format allows you to fill any attribute with any key from the querystring. To use multiple keys from the querystring you can put a comma-separated list of these expressions (for example: qs="key1=attribute1,key2=attribute2"). If you need an attribute to be added when a querystring key is not present, replace the "=" with "!=" (for example: qs="key1!=attribute1", attribute1 will only be added if key1 is not in the querystring). As a shorthand, if the key in the querystring is the same name as the attribute you want to fill, you can omit the attribute name in the expression (for example: qs="key1="). If an expression doesn't find it's key in the querystring: it reverts the attribute that it fills to it's original value.
  • qs="key". This format will update the [value] attribute with the querystring whenever the querystring changes.
Attributes [date-placeholder=""] and [time-placeholder=""]:

The [date-placeholder=""] applies the [placeholder=""] to the date component of the timestamp

The [time-placeholder=""] applies the [placeholder=""] to the time component of the timestamp

Attribute [mini]:

Sometimes you are limited on space. The [mini] attribute can be used to make the <gs-date> smaller.

Attributes [no-date-picker] and [no-time-picker]:

If you want to remove the date picker button, simply add the [no-date-picker] attribute.

If you want to remove the time picker button, simply add the [no-time-picker] attribute.

Attributes [date-tabindex=""] and [time-tabindex=""]:

The [date-tabindex=""] applies the [tabindex=""] to the date component of the timestamp

The [time-tabindex=""] applies the [tabindex=""] to the time component of the timestamp

Attribute [disabled]:

With the [disabled] attribute, you can control whether or not you can use the control.

Attribute [readonly]:

The [readonly] attribute is similar to [disabled] because it prevents the value from changing. The differences are that readonly is styled differently and allows you to copy the current text from the control.

Attribute [date-format]:

The [date-format] attribute describes how you want the date component to display.

The following format names are accepted:

Name Example
01/01/2015
shortdate 1/1/15
mediumdate Jan 1, 2015
longdate January 1, 2015
fulldate Thursday, January 1, 2015
isodate 2015-01-01
isodatetime 2015-01-01T00:00:00

You can specify your own format with these specifiers:

Name Output
y Full year
yyyy Full year
yy Last two digits of the year
M Month number
MM Padded month number
MMM Short month name
MMMM Full month name
d Day of the month
dd Padded day of the month
EEE Short day name
EEEE Full day name
H Hour number (24-hour, midnight 0)
HH Padded hour number (24-hour, midnight 0)
h Hour number (12-hour)
hh Padded hour number (12-hour)
K Hour number (24-hour, midnight 12)
KK Padded hour number (24-hour, midnight 12)
AM AM/PM
m Minutes
mm Padded minutes
s Seconds
ss Padded seconds
S Milliseconds
SS Padded milliseconds (length 2)
SSS Padded milliseconds (length 3)

NOTE: If you have a time in your date format, arrowing will not work in the date component.

Attribute [time-no-now-button]:

If you want to remove the "Now" button from the time picker: add the "time-no-now-button" attribute.

Attribute [time-non-empty]:

If you want to force the user to input a time value: add the non-empty attribute. Note: if a default value is not provided then the control defaults to "12:00 PM".

Attribute [time-format]:

The [time-format] attribute determines the display format of the control. This attribute can take two different values: "regular" and "military". "regular" is normal 12-hour time and "military" is 24-hour time.
