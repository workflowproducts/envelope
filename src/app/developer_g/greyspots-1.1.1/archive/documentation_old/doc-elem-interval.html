Interval

Tag:

<gs-interval></gs-interval>

Description:

The <gs-interval> control is like the <gs-text> control, but it is specialized for amounts of time.

Examples:

Skeleton Example:

A basic <gs-interval> element. Without attributes the <gs-interval> control will have the following behaviors:

  1. There is a button that activates the time picker.
  2. After choosing a time with the time picker, the time picker disappears and the data changes in the control.
  3. You can also change the time manually.
  4. The [value=""] attribute will contain the amount of time with minute precision.
Attribute [column]:

When you have a gs-interval inside a data element and you want to use it to display cell data you use the column attribute.

Attribute [value=""]:

To set the value for the <gs-interval>, use the [value=""] attribute.

Attribute [unit=""]:

The <gs-interval> defaults to treating it's value as a number of minutes. However, you may need to more or less precision than that. To store using hour or second precision you need to adjust the unit that this control uses to translate/store it's value in. To do this you can use the [unit=""] attribute.

Attribute [hour-places=""]:

To change the number of hours that can be shown in the <gs-interval> use the [hour-places=""] attribute.

Attribute [onchange=""]:

To run javascript when the value changes you can use the [onchange=""] attribute.

Attribute [qs=""]:

The name [qs] stands for "querystring". The querystring is the part in the URL after the "?". The [qs] attribute has two formats, each with their own use.

  • qs="key1=attribute1". This format allows you to fill any attribute with any key from the querystring. To use multiple keys from the querystring you can put a comma-separated list of these expressions (for example: qs="key1=attribute1,key2=attribute2"). If you need an attribute to be added when a querystring key is not present, replace the "=" with "!=" (for example: qs="key1!=attribute1", attribute1 will only be added if key1 is not in the querystring). As a shorthand, if the key in the querystring is the same name as the attribute you want to fill, you can omit the attribute name in the expression (for example: qs="key1="). If an expression doesn't find it's key in the querystring: it reverts the attribute that it fills to it's original value.
  • qs="key". This format will update the [value] attribute with the querystring whenever the querystring changes.
Attribute [mini]:

Sometimes you are limited on space. The [mini] attribute can be used to make the <gs-interval> smaller by removing the borders and padding.

Attribute [no-picker]:

If you want to remove the time picker button, simply add the [no-picker] attribute.

Attribute [tabindex=""]:

With the [tabindex=""] attribute, you can control what order your controls are tabbed to.

Attribute [disabled]:

With the [disabled] attribute, you can control whether or not you can use the control.

Attribute [readonly]:

The [readonly] attribute is similar to [disabled] because it prevents the value from changing. The differences are that readonly is styled differently and allows you to copy the current text from the control.

Javascript Methods:

These functions can be run on individual <gs-interval> elements:
INTERVAL.open() <-- This method opens the <gs-interval>.
