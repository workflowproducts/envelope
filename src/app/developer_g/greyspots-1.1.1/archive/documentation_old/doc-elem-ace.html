Ace

Tag:

<gs-ace></gs-ace>

Description:

The purpose of a <gs-ace> control is to hold large amounts of text from the user.

Examples:

Skeleton Example:

A basic <gs-ace> element. Without attributes the <gs-ace> control will have the following behaviors:

  1. A scrollbar will appear when the text is longer then the <gs-ace>.
Attribute [column]:

When you have a gs-ace inside a data element and you want to use it to display cell data you use the column attribute.

Attribute [mode]:

gs-ace has syntax coloring

Attribute [rows]:

The rows attribute sets the height of the gs-ace in rows.

Attribute [onchange=""]:

To run javascript when the value changes, use the [onchange=""] attribute.

Attribute [value=""]:

To set the default value for the <gs-ace>, use the [value=""] attribute.

Attribute [encrypted="keyVariable"]:

When you have a database column that contains sensitive information use the [encrypted] attribute to encrypt the value. The column will be encrypted with AES using the global variable that matches the [encrypted] attribute. If the gs-ace detects a bad password, it will emit a password-error event with the key variable in event.keyVariable and either 'bad' or 'no' in event.reason. The password for this example is 'test'.

Attribute [qs=""]:

The name [qs] stands for "querystring". The querystring is the part in the URL after the "?". The [qs] attribute has two formats, each with their own use.

  • qs="key1=attribute1". This format allows you to fill any attribute with any key from the querystring. To use multiple keys from the querystring you can put a comma-separated list of these expressions (for example: qs="key1=attribute1,key2=attribute2"). If you need an attribute to be added when a querystring key is not present, replace the "=" with "!=" (for example: qs="key1!=attribute1", attribute1 will only be added if key1 is not in the querystring). As a shorthand, if the key in the querystring is the same name as the attribute you want to fill, you can omit the attribute name in the expression (for example: qs="key1="). If an expression doesn't find it's key in the querystring: it reverts the attribute that it fills to it's original value.
  • qs="key". This format will update the [value] attribute with the querystring whenever the querystring changes.
Attribute [autoresize]:

Rather then automatically resizing the <gs-ace> to take up 100% space, it can automatically resize itself when you use the [autoresize] attribute to the text length.

Attribute [disabled]:

With the [disabled] attribute, you can control whether or not you can use the control.

Attribute [readonly]:

The [readonly] attribute is similar to [disabled] because it prevents the value from changing. The differences are that readonly is styled differently and allows you to copy the current text from the control.
