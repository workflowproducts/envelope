Memo

Tag:

<gs-memo></gs-memo>

Description:

The purpose of a <gs-memo> control is to hold large amounts of text from the user.

Examples:

Skeleton Example:

A basic <gs-memo> element. Without attributes the <gs-memo> control will have the following behaviors:

  1. You can resize the <gs-memo>.
  2. A scrollbar will appear when the text is longer then the <gs-memo>.
Attribute [column]:

When you have a gs-memo inside a data element and you want to use it to display cell data you use the column attribute.

Attribute [rows]:

The rows attribute sets the height of the gs-memo in rows.

Attribute [allow-tab-char]:

A default <gs-memo> does not allow tab characters. To allow tab characters add the [allow-tab-char] attribute.

Attribute [onchange=""]:

To run javascript when the value changes, use the [onchange=""] attribute.

Attribute [value=""]:

To set the default value for the <gs-memo>, use the [value=""] attribute.

Attribute [qs=""]:

The name [qs] stands for "querystring". The querystring is the part in the URL after the "?". The [qs] attribute has two formats, each with their own use.

  • qs="key1=attribute1". This format allows you to fill any attribute with any key from the querystring. To use multiple keys from the querystring you can put a comma-separated list of these expressions (for example: qs="key1=attribute1,key2=attribute2"). If you need an attribute to be added when a querystring key is not present, replace the "=" with "!=" (for example: qs="key1!=attribute1", attribute1 will only be added if key1 is not in the querystring). As a shorthand, if the key in the querystring is the same name as the attribute you want to fill, you can omit the attribute name in the expression (for example: qs="key1="). If an expression doesn't find it's key in the querystring: it reverts the attribute that it fills to it's original value.
  • qs="key". This format will update the [value] attribute with the querystring whenever the querystring changes.
Attribute [placeholder=""]:

The [placeholder=""] attribute can be used when you don't want to use a label to explain the purpose of the control to the user. (The label and the placeholder can be used together, that's just not what normally happens.)

Attribute [autoresize]:

Rather then manually resizing the <gs-memo> when the text gets too large, it can automatically resize itself when you use the [autoresize] attribute.

Attribute [no-resize-handle]:

When you use the [no-resize-handle] attribute, you can no longer resize the memo. When you use both the [autoresize] attribute, you typically also want [no-resize-handle].

Attribute [mini]:

Sometimes you are limited on space. The [mini] attribute can be used to make the <gs-memo> smaller.

Attribute [tabindex=""]:

With the [tabindex=""] attribute, you can control what order your controls are tabbed to.

Attribute [disabled]:

With the [disabled] attribute, you can control whether or not you can use the control.

Attribute [readonly]:

The [readonly] attribute is similar to [disabled] because it prevents the value from changing. The differences are that readonly is styled differently and allows you to copy the current text from the control.
