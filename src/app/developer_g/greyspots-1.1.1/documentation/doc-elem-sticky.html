Sticky

Tag:

<gs-sticky>
    <gs-sticky-inner>...</gs-sticky-inner>
</gs-sticky>

Description:

The gs-sticky element is for headers or footers that want to be stuck to the top or bottom of the screen only when they are scrolled past. Note 1: the gs-sticky element must have it's contents in a gs-sticky-inner. Note 2: in these examples the gs-sticky-inner elements have the "bg-primary" attribute on them just to make them easier to see.

Examples:

Skeleton Example:

A basic gs-sticky element. Without attributes the gs-sticky will have the following behaviors:

  1. On a non-touch device: when the window is scrolled down below the point where the element naturally wants to be, it will stick to the top of the window
Attribute "direction":

The "direction" attribute allows you to make the contents of the gs-sticky element stick to the bottom. This is done by setting the "direction" attribute to "down". Note: scroll all the way down to see the code and scroll all the way down to see the footer's behaviour.

Attribute "stuck":

The "stuck" attribute allows you to make the contents of the gs-sticky element always stick to the top or bottom depending on your choice. This is done by adding the "stuck" attribute.

Attribute "touch-device-allowed":

The "touch-device-allowed" attribute makes it so that the contents of the gs-sticky element stick on touch devices as well as non-touch devices. This is done by adding the "touch-device-allowed" attribute.
