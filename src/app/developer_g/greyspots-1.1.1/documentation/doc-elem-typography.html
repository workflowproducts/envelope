Typography

Description:

These classes and tags are used to convey information without you having to worry about making everything look nice.

Examples:

Headers:

Headers by default have padding. The <small> tag can be used for secondary text. Using a span with a class .h1 to .h6 gives you the same text styling, but no padding.

Paragraph:

Paragraphs, (<p> tags) somtimes have a header right above.

Leading Paragraph:

Use the .lead class to make a paragraph stand out from the other paragraphs.

Inline Text Modifiers:

There are several modifiers to the text that come from html5.

Horizontal Rule:

Use the <hr/> tag make a logical seperation between sections of the page.

Text Alignment:

Use the these classes to align the text.

  • The .text-left class aligns the text to the left.
  • The .text-right class aligns the text to the right.
  • The .text-center class centers the text.
  • The .text-justify class increases and decreases the space between words so the line always ends at the right edge.
  • The .text-nowrap class prevents wrapping of the text.
Text Transformation:

Use the these classes to tranform the text.

  • The .text-lowercase class makes the entire text lowercase.
  • The .text-uppercase class makes the entire text uppercase.
  • The .text-capitalize class capitalizes the first letter of each word.
Background and Text Colors:

Use these attributes to change the color of the background and text. It's important not to get caught up in the colors too much, use the name of the attribute as its purpose. Use [bg-danger] or [txt-danger] for error messages and possible problems. (Like deleting a file) Use [bg-warning] and [txt-warning] for warnings, etc..

  • The [txt-muted] attribute makes the text gray.
  • The [txt-primary] attribute makes the text dark blue.
  • The [txt-success] attribute makes the text green.
  • The [txt-info] attribute makes the text light blue.
  • The [txt-warning] attribute makes the text orange.
  • The [txt-danger] attribute makes the text red.
  • The [bg-muted] attribute does not exist.
  • The [bg-primary] attribute makes the background dark blue.
  • The [bg-success] attribute makes the background green.
  • The [bg-info] attribute makes the background light blue.
  • The [bg-warning] attribute makes the background yellow.
  • The [bg-danger] attribute makes the background red.
Table Background Colors:

Use these attributes to change the color of the background. It's important not to get caught up in the colors too much, use the name of the attribute as its purpose.

  • The [bg-success] attribute makes the background green.
  • The [bg-warning] attribute makes the background yellow.
  • The [bg-danger] attribute makes the background red.
  • The [bg-info] attribute makes the background light blue.
Abbreviations:

Use the <abbr> tag to explain an abbreviation. Put the full text of the abbreviation in the [title] attribute. Use the .initialism class to make the abbreviation uppercase and a little smaller then the rest of the text. When the user hovers over the abbreviation, it will show the full text.

Addresses:

Use the <address> tag to wrap contact information.

Blockquote:

Use the <blockquote> tag to wrap a quote. Use the <small> tag or preferably the <footer> tag for the attribution. Cite the source with the <cite> tag and the [title] attribute. Use the .pull-right class to align the quote to the right.

Lists:

Use the standard <ul> tag for a unordered list, the <ol> tag for ordered lists, and the <li> tag for items. The .list-unstyled class removes the extra styling around that list, but not for sub-lists. The .list-inline class will put all the items on the same line.

Lists:

Use the standard <dl> tag for a list of terms. The <dt> tag is for term names, and the <dd> tag is term descriptions. The .dl-horizontal class puts the term on a column to the left, and the descriptions to the right. However, if the screen is too small it will go back to the original styling.

Code:

There are several tags you can use to format code. You can nest them, but you don't have to.

  • Use the <code> tag for an inline snippet.
  • Use the <kbd> tag for a command you should type.
  • Use the <pre> tag for a block of code.
  • Use the <var> tag for a variable, such as in an equation.
  • Use the <samp> tag for a block of output.
