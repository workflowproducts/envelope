Folder Element

Tag:

<gs-folder></gs-folder>

Description:

The gs-folder tag enables you to navigate folders, upload files, delete files, rename files and open the file editor for files.

A basic gs-folder element:

Attribute [path=""]:

The [path] attribute is for defining what directory the element should start in. Users cannot navigate to folders above the [path=""] setting (this is merely a front-end limitation, do not depend on this as a security measure).

Attribute [horizontal=""]:

The [horizontal] attribute is for putting the file list and the folder list side by side.

Attribute [no-folders=""]:

The [no-folders] attribute is for hiding the folder list. This can be used to prevent the user from navigating anywhere other than the folder they start in (this is merely a front-end limitation, do not depend on this as a security measure).

Attribute [no-files=""]:

The [no-files] attribute is for hiding the file list.

Attribute [mini=""]:

The [mini] attribute removes the border around the element.

Attribute [qs=""=""]:

The name [qs=""] stands for "querystring". The querystring is the part in the URL after the "?". The [qs] attribute has two formats, each with their own use.

  • qs="key1=attribute1". This format allows you to fill any attribute with any key from the querystring. To use multiple keys from the querystring you can put a comma-separated list of these expressions (for example: qs="key1=attribute1,key2=attribute2"). If you need an attribute to be added when a querystring key is not present, replace the "=" with "!=" (for example: qs="key1!=attribute1", attribute1 will only be added if key1 is not in the querystring). As a shorthand, if the key in the querystring is the same name as the attribute you want to fill, you can omit the attribute name in the expression (for example: qs="key1="). If an expression doesn't find it's key in the querystring: it reverts the attribute that it fills to it's original value.
  • qs="key". This format will cause the gs-folder to refresh when a change occurs in the query string and the key is present.
Attribute [no-list-error=""]:

The [no-list-error] attribute causes errors that occur when the gs-folder attempts to load a folder to be ignored. This attribute is useful because sometimes, the folder that a gs-folder is looking at may not exist yet. Note: Once the user uploads or creates a file inside the folder: the necessary folders will be created automatically.

Attribute [disabled=""]:

The [disabled] attribute causes the gs-folder to disallow changes.
