Page

Tag:

<gs-page>     <gs-header>...</gs-header>     <gs-body>...</gs-body>     <gs-footer>...</gs-footer> </gs-page>

Description:

These elements provide desktop and mobile aware containers for your applications. Headers and footers are fixed at the top and bottom of the viewport and expand to display whatever you put in them. The gs-body contains your main content and becomes scrollable when appropriate.

Examples:

Skeleton Example:

A basic gs-page element with a gs-body. The page will expand to 100% width and height and if the content is still too long to be displayed, the gs-body will get a scrollbar.

Header Example:

A gs-page element with a gs-body and a gs-header.

Note the header will be fixed to the top of the gs-page and if the body content is too long it will get a scrollbar.

Footer Example:

A gs-page element with a gs-body and a gs-footer. The footer will be fixed to the bottom of the gs-page and if the body content is too long it will get a scrollbar.

Header And Footer Example:

gs-page element with a gs-body, gs-header and a gs-footer. The header and footer will be fixed to the top and bottom respectively and if the content is too long the gs-body will get a scrollbar.

Header And Envelope Example:

A gs-page element with a gs-body (containing a gs-envelope) and a gs-header.

Header And Footer Button Example:

A gs-page element with a gs-body, gs-header and a gs-footer. In this example the header and footer each have a grid of buttons inside them.
